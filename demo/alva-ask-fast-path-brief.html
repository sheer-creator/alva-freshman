Product Brief

Alva Ask 轻量化方案

小白和轻度用户进入 Alva 后,看到价值的速度太慢。ask 场景在 harness 层没有独立的轻量路径,启动时加载完整 builder context,模型默认倾向先 build 而非先给结论。

00

现状(代码事实)

  • 7 种 session kind 全部加载同一个 alva skill(profiles.ts);ask 的 ASK.md overlay 当前为空 placeholder,对 ask 没有任何差异化。
  • 渲染时全量注入 1634 行 SKILL.md(buildSkillSection),ask 与 build 拿到的 context 基本一致。
  • answer-first / 防过度 build 的约束(Reuse Before Build、Artifact Consent)只存在于运行时 prompt,profile 层不做分流,轻重判断的成本压在模型读完全量 context 之后。
  • 首页 suggested prompt 携带 skillLanding,点击即带 skill 提交(RecommendPrompts.tsx),入口本身把用户推向 build。
01

填实 ASK.md

  • 改哪里:config/profiles/ASK.md,当前 body 为空。body 渲染成 “Session Profile: Ask” 段,append 在 base CLAUDE.md 和全量 SKILL.md 之后。位置在最后,指令权重高,能压制前面的 builder 倾向。
  • 写什么:默认直接回答数据 / 研究 / 比较类问题;one-off 图表走 RenderChart 返回 chat;playbook、automation、alert 等持久产物只在用户明确要求时做,动手前先确认;答完后用一句话提示升级路径。
  • 局限:overlay 是叠加指令,删不掉已注入的 builder context,只能纠正模型倾向,降不了启动开销。减重靠第 2 步。
02

拆 alva-ask 轻量 skill

  • 改哪里:新增 alva-ask skill,ask profile 改挂 skills: ['alva-ask']。buildSkillSection 按 skill 全量拼接,换 skill 才能真正减小注入体量。
  • 保留:精简 Pre-flight(whoami / memory 走 <session-prefill>)、数据查询(alva CLI、data-skills、search)、RenderChart 出图、language.md voice、Reuse Before Build。
  • 去掉:release / deploy / feed 写作工作流、Altra 回测引擎细节、design-components / design-widgets(合计 3000+ 行)、playbook publish / remix 全流程。
  • 边界(需拍板):ask 里用户确实想 build 时,引导切到 build 路径,而非在 ask skill 里塞回 build 能力。
03

suggested prompt 入口纠偏

  • 前端(RecommendPrompts.tsx):点击 prompt 默认填入输入框,不自动提交;skill 作为可选附加,不默认注入 <use-skill>。
  • 后端:收紧 skillLanding 生成策略,只在 prompt 本身是明确 build / remix 意图时才带 skill。
  • 待确认:后端 suggestedPrompts 生成逻辑与前端 onSelect 提交链路需先确认现状。

4a · 画像初始化收集(GUI)

05

build 引导(ask → playbook 转化)

build 不做首屏首推,放在 ask 出成果之后的链路上,两个触发点。

5a · 对话内即时建议

ask 给出成果后,判断这次结果是否值得沉淀(可持续跟踪的标的 / theme / 信号),在回复末尾用一句话建议转成 playbook,用户点头再做。触发收敛:一次性结论不提示,带“持续跟踪 / 定期更新”价值的才提示。

GUI 示意 · 对话内转化点

你 NVDA 这波 AI infra 行情还能跟吗?
A 动能仍在,但估值已偏高。近 1 个月 +14%,资金面与新闻面正向,P/E 58 处于历史高位区间,回调风险集中在财报前后。 想让它每天自动更新、有变化提醒你? 转成 Playbook

5b · proactive 召回

通过 Alva Agent 主动消息(heartbeat / timeline),把用户最近多次关注的标的与 theme(来自 ask 历史与 memory user.md)接回构建链路,两种力度:

  • 建议构建:挑出重复出现的几个标的,提示可以做成自动更新的 playbook,用户一键进入。
  • 静默预建:Alva 在后台直接把 playbook 建好,不消耗用户 credits,完成后退给用户查看,用户决定保留、继续打磨或丢弃。
