Product Spec + Visual Demo

Alva Channel

A single-page product spec for turning personal and group surfaces into one durable investment workspace model.

Alva Channel is where Alva remembers, tracks, and works with you or your group. Memory remembers. Playbooks run. Chats and Tasks move work forward. Artifacts preserve what Alva generated or processed.

Model

对象定义

Object Meaning
Channel 由 Alva Agent 维护的上下文空间，scope 可以是 personal 或 group。
Memory Channel 的长期记忆。Personal 是个人偏好和历史决策；Group 是群共识、关注对象、偏好和已验证/失效 thesis。
Playbooks Channel 正在创建、订阅、跟踪、讨论或使用的投资应用。
Chats / Tasks Channel 内发生的对话、research task、agent task、scheduled task 和需要用户输入的工作。
Artifacts Alva 生成或处理过的文件、报告、数据、source packs、skills 和策略草稿。
Endpoint Web、Telegram DM、Discord DM、Telegram group、Discord group、Mini App。
下方原型里的四个 tabs 对应这个对象模型：Memory / Playbooks / Chats/Tasks / Artifacts。
User Stories

用户故事

群管理员 / 社群主理人

我想把 Alva 加进 Telegram 或 Discord 群，让群里的讨论、KOL 观点、playbook、digest 规则沉淀成一个共享 Channel，而不是每天从聊天记录里重新捞信息。

Goal: group intelligence layer

已绑定 Alva 的群成员

我想在群里直接问 Alva、保存重要结论、让它持续 track 某个 ticker/KOL/playbook，并能回到 web 页面查看完整上下文和产物。

Goal: ask, save, track

未绑定账号的群访客

我先浏览这个 group channel 的公开 Memory、tracked playbooks 和最新 brief，判断这个群的投研质量，再决定是否绑定 Alva 账号参与写入。

Goal: browse before bind

Playbook 创建者 / KOL campaign 运营者

我想把一个 playbook 分发到相关 group channel，让它成为群里的长期跟踪对象，并通过群成员反馈、订阅和讨论获得冷启动流量。

Goal: distribute with context

个人投资者

我希望群里的公共结论能帮助我，但涉及 portfolio、交易账户、private files 和个人偏好时，Alva 必须切回 My Alva，不能把私人上下文带进群。

Goal: public signal, private action

Alva Agent

我需要在群聊、mini app 和 web 页面之间保持同一个 Channel state：群里轻量回应，web 上组织证据、任务、文件和长期 Memory。

Goal: one state, many endpoints
Journey

用户旅程

01 群里触发
02 打开页面
03 绑定写入
04 Agent 维护
05 私域 handoff
1. 群里发现 Alva 用户在 TG/DC 群里看到 Alva 的 digest、material change alert、KOL 回测卡片或 `/alva track` 命令结果。
2. 打开 canonical page 群消息、mini app 或 Discord card 指向同一个 Alva Group Channel 页面，访客能先浏览公开上下文。
3. 绑定后参与写入 绑定 Alva 账号后，用户可以 Ask、Save to Memory、上传 source、启动 task 或让 Alva track 新对象。
4. Agent 持续维护 Alva 把短期聊天 buffer、群共识、playbook 输出和 source packs 处理成 Memory、Tasks 和 Artifacts。
5. 公共到私人 handoff 当问题涉及仓位、交易、private memory 或个人文件时，Alva 从 Group Channel 切回 My Alva 执行。
核心转化不是“群聊里多一个 bot”，而是把群聊里的投研共识变成可浏览、可复用、可持续更新的 Alva workspace。

record / workspace

Web / Mini App: system of record

  • 承载完整 Channel 页面：Memory、Playbooks、Chats/Tasks、Artifacts。
  • 适合浏览、检索、筛选、验证来源、查看回测和处理文件。
  • 负责权限边界：访客只读，绑定成员可写，admin 可配置 digest 和 tracked targets。
  • 处理复杂工作流：长任务、文件上传、playbook remix、private handoff、portfolio/trading 操作。

Telegram / Discord: activation surface

  • 承接群里的自然讨论，在用户已经聚集的地方触发 Alva。
  • 适合轻量命令、mention、reply、digest、material change alert 和绑定引导。
  • 默认不把所有聊天都写入 Memory，只在明确保存或高置信判断时沉淀。
  • 负责把用户带回 canonical Channel page，而不是在聊天窗口里塞完整产品。

定位

群里是“入口和触发器”，Web 是“状态和工作台”。Alva 不试图替代 Telegram/Discord，而是在群之上提供一层投资智能和长期记忆。

增长含义

每个 group channel 都可以变成一个公开或半公开的投研页面：访客可浏览，绑定用户可贡献，playbook 可分发，agent 可持续更新。

群组联动流程

  • Invite Alva用户把 Alva 加到 TG/DC 群后，系统创建 canonical Group Channel page。
  • Open Channel群消息卡片、Telegram mini app、Discord card 都指向同一个 Channel 页面。
  • Browse first群成员和访客可以浏览公开 Channel 内容，先看到 Memory 摘要、tracked playbooks 和最新 brief。
  • Bind to write绑定 Alva 账号后，成员可以 Ask Alva、Save to Memory、Track target、上传 source 或启动 task。
  • Material changes onlyAlva 回群必须克制：mention、command、reply、scheduled digest 或 material change 才出现。
Boundaries

权限和隐私边界

Role Capability
Guest 浏览公开 Channel 页面；不能写入 Memory 或启动任务。
Unbound member 在群里看到 Alva 消息；点击链接浏览；收到绑定引导。
Bound member Ask、Save to Memory、Track public targets、提交 source、参与 task。
Group admin 管理 tracked targets、digest cadence、visibility、成员写权限和 group-level artifacts。
Alva Agent 整理、验证、更新、提醒；但不能把个人 memory、portfolio 或 private files 暴露给群。
