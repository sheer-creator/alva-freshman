Alva · Mobile concept · v4

抽屉导航 · 进频道即首屏 · 频道带只读 context · 对话外置 Telegram

移动端无底部导航（左侧抽屉），打开直接落在某个频道（与 PC 落到内容一致，抽屉就是频道列表）。每个频道/群聊都有 Feed / Memory / Tasks / Artifacts 只读上下文。新用户在频道里用「预设 KOL 合集」批量勾选，点 Ask Alva in Telegram 把名单 + 连接指令带进 Telegram 完成 build。v4 对齐 Web 端迭代：所有推送标注出自哪个 Automation；BTC 群改为 Room-Digest 摘要即对话（raw 镜像归 DeFi）；Artifacts 含只读 Automations；补未连接首屏态。

Alva 原生（teal）— 浏览 / 内容 Telegram — 对话 / build 入口 Discord — 次选
01

导航与首屏 · 进频道即首页

无底部 Tab，左侧抽屉沿用 PC 侧边栏。建议默认直接落进某个频道（详情）——抽屉本身即频道列表，和 PC「落到内容、侧栏导航」一致；PC 暂无独立的频道列表页，所以列表页在移动端作为可选项。首屏 app bar 带 Alva 品牌。

02

新用户 Onboarding · 预设 KOL 合集，批量勾选

新用户进到 KOLs · AI Digest：顶部先看到一条真实推送感受价值，下面用「预设 KOL 合集」批量勾选（不再选主题，只针对 KOL），也能加自己的 @handle。点「Ask Alva in Telegram」把所选合集 + 连接账号指令发到 Telegram，在 IM 里完成 build。

03

群聊 Channel · 只读，但能读全部 context

与 Web 端两种形态对齐：BTC = Room-Digest 摘要即对话，DeFi = raw 只读镜像，发言一律跳进群。Memory / Tasks / Automations / Artifacts 上下文都能在 app 里读 —— 浏览无门槛。

04

Playbook 详情 · 边浏览边想聊

底部只保留一个清晰的主操作 「Ask Alva about this」（修掉了之前溢出的按钮、也去掉了和它重复的 Telegram 按钮）。点它弹出连接 sheet —— 选 Telegram / Discord、说明会把这个 Playbook 的上下文带进聊天，并能追问、订阅提醒、转发到群。
