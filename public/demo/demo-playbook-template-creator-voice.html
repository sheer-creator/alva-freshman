Creator Voice Surfaces

Playbook Template 的说明信息分层

极简示意：把 Creator 的主观解释放到不同可见度的产品表面，避免所有话都挤进 iframe 或每个 widget。

Core rule
越显眼，越短；越复杂，越折叠。
默认阅读路径必须清楚，长解释不能成为理解 Playbook 的前置条件。
Container - desc&readme
容器说清楚“这是什么”。
Description 给 3 秒理解，README 给完整机制和作者意图。
iFrame - tab
整页方法论可以成为独立 Tab。
最重的 how / why / source 说明，放在和 Thesis、Basket 平级的 Methodology。
iFrame - widget
页面内只强调必须看到的判断。
Widget 大块说明只用于关键论点、反常识结论或风险提示。
iFrame - tooltip
复杂口径放在就近折叠层。
指标定义、数据来源、引用 claim、盲点，用 tooltip / popup 随点随看。

Default reading path stays clean

模板默认页面应该靠标题、第一屏、图表和短说明让用户直接读懂。长方法论、数据细节和 caveat 不该塞满主路径。

原则：如果没有强烈理由，不要把每个 widget 都变成说明文。

Methodology tab: when the explanation is the product surface.

这是最重的 iframe 内说明方式：和 Thesis、Basket 平级，承载完整方法论、数据来源、限制和使用说明。适合用户必须能系统阅读，但不应该默认塞进每个 widget 的内容。

full-tab explanation

Mechanism

解释 feed、ADK narrative、评分链路，以及哪些内容由同一记录生成。

Data sources

列真实调用的数据源、刷新频率、失败时的降级或空态策略。

How to use

说明适合看什么信号、如何解读变化、何时只看推送即可。

Blind spots

把修订数据、私有合同、样本偏差等限制放在完整语境里。

Info tooltip / popup

复杂指标、表格列、引用 claim、数据刷新口径和局部 caveat 放在这里。默认折叠，不打断主阅读路径。

README: How this Playbook works

外层深阅读入口。默认由 Alva skill 生成，未来可以支持 Creator editor。

Why

说明作者为什么认为这个主题值得持续追踪，以及它和机会、风险、价格之间的关系。

What

列出 Playbook 的主要产出：核心结论、指标、图表、来源、推送和使用场景。

How

解释 feed、数据源、ADK narrative、计算口径和更新链路；只声明真实调用的数据源。

When

说明刷新频率、适合什么阶段、何时看推送，何时打开页面深读。

Information hierarchy Description 负责识别；README 负责外层完整解释；Methodology Tab 负责 iframe 内最重说明；Widget 负责必须看到的观点；Tooltip 负责局部复杂口径。
